# Official Orange brand assets

Status: official project emblem, wordmark, and lockup assets designated by the
Bootstrap Steward on 2026-07-11

These eight images were imported byte-for-byte from the steward-supplied
`Orange-Assets` collection. The machine-readable [`manifest.json`](manifest.json)
records each source filename, dimensions, transparency, role, and SHA-256
digest. The Gate 0 repository policy independently admits the same exact paths
and digests.

`orange.png` and `orange-banner.png` retain embedded C2PA Content Credentials
that identify OpenAI Media Service API generation using `gpt-image 2.0` and the
`trainedAlgorithmicMedia` assertion. Preserving the original bytes preserves
that provenance evidence. The remaining supplied variants do not carry the
same signed manifest, so visual similarity alone must not be presented as
equivalent signed provenance.
The embedded claim and certificate material were inspected, but their
cryptographic validity has not yet been independently verified.

## Preferred assets

- [`orange.png`](orange.png) is the primary emblem for a light background.
- [`IMG_2768.PNG`](IMG_2768.PNG) is the primary transparent emblem.
- [`orange-banner.png`](orange-banner.png) is the primary horizontal emblem and
  wordmark lockup for a light background.
- [`IMG_2766.PNG`](IMG_2766.PNG) is the transparent wordmark.

The JPEG files are compact opaque alternatives. The UUID-named PNG files and
`IMG_2760.JPEG` are preserved official source variants. In particular,
`4DB7A71A-8FF8-48B3-8243-1657017AD816.PNG` has the same decoded pixels as
`orange.png`, but its distinct original encoding and digest are retained.

## Integrity and rights boundary

Do not optimize, recolor, crop, rename, or replace these originals in place.
Add a clearly named derived asset, record its source and transformation, and
update both the manifest and repository policy instead. This preserves a stable
review and provenance boundary for visible project identity.

They are the official working Orange identity assets for this repository.
Their inclusion authorizes that current repository use.
It does not resolve the project-name and trademark decision or create a
repository-wide copyright or redistribution license. Those boundaries remain
subject to D-017 and D-018 in the [decision register](../../docs/DECISIONS.md).
